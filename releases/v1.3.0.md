# agent-workflow-kit v1.3.0 — portable AI-agent memory & workflow, now aligned with Anthropic's Skills guidance

**Cross-agent memory and workflow for AI coding agents — Claude Code, Codex, Cursor, Devin Desktop, GitHub Copilot.** One command deploys an `AGENTS.md` entry point + a `docs/ai/` context system with cap / archive / index enforcement into any repository. v1.3.0 is a kernel-authoring refinement release: it sharpens how the skill itself is written, following Anthropic's [*Lessons from building Claude Code: how we use Skills*](https://claude.com/blog/lessons-from-building-claude-code-how-we-use-skills).

```bash
npx @sabaiway/agent-workflow-kit@latest init
```

Then, in any repo: `/agent-workflow-kit` to bootstrap, or `/agent-workflow-kit upgrade` to migrate an existing deployment.

---

## What's new in 1.3.0

This release changes **how the kernel is authored**, not what gets deployed into your project — so **no migration is required**. Running `/agent-workflow-kit upgrade` on a 1.1.x / 1.2.x project simply reconciles and re-stamps to `1.3.0` with nothing to apply.

- **Consolidated Gotchas section.** Anthropic's guidance calls the Gotchas section *"the highest-signal content in any skill."* The non-obvious traps — the harness-added `Co-Authored-By` trailer vs. prose, hidden mode never touching `package.json`, `CLAUDE.md` as a symlink (not a copy), source-vs-target directory, no-Node → skip enforcement, never overwriting an existing entry point or git hook — are now a single scannable list in `SKILL.md`.
- **Progressive disclosure for setup contracts.** The full Visibility / Communication / Attribution rules moved to `references/contracts.md`; `SKILL.md` keeps a lean pointer with one-line defaults. The always-loaded skill file is ~40 lines lighter, and the detail loads only when an agent actually needs it — the file-system-as-context-engineering pattern from the blog.
- **Structured setup prompts.** The three bootstrap questions (visibility, conversational language, agent attribution) and their `upgrade` equivalents now call for a structured multiple-choice prompt (`AskUserQuestion` in Claude Code) where the agent supports it, falling back to prose elsewhere — keeping cross-agent portability (Codex / Cursor / Devin Desktop) intact.

## Why it matters

If you run more than one AI coding agent, or hand work between sessions, context evaporates between runs. agent-workflow-kit gives any agent — including a fresh session of the same one — a way to reconstruct project context in about 60 seconds, find the current task, and avoid repeating past mistakes. It is stack-agnostic, English-only for cross-team portability, and ships its enforcement as dependency-free Node scripts with a pre-commit hook.

## Install & upgrade

| Goal | Command |
|------|---------|
| Install or refresh the kit | `npx @sabaiway/agent-workflow-kit@latest init` |
| Bootstrap a repo (first time) | `/agent-workflow-kit` |
| Upgrade an existing deployment | `/agent-workflow-kit upgrade` |

Requirements: Node ≥ 18. Works with Claude Code, Codex, Cursor, Devin Desktop, GitHub Copilot, and any agent that reads the `AGENTS.md` open standard.
